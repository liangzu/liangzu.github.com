Computing has become a ubiquitous concept since the advent of electronic computers. After several decades of development in all fields that involve computing, people studying programming nowadays are embracing a set of tools of remarkable power. At the same time, it raises a question: what is the best way to go about learning to program?

The way to learn depends on the goal of the learner, perhaps as well as how his or her goal is perceived by the instructor. One goal is to study with a purpose; I want to implement a data analysis algorithm, I want to process this excel file, I want to send emails automatically. In this case, one already has a problem to solve, knows that programming helps, and desires to learn it. Another goal is to let programming be one's starting point in order to explore the computing world. This demands a solid ground so as to soar higher. Two different goals give rise to two different strageties. To meet the first goal one seeks what oneself needs in the computing world to solve the problem, perhaps no more than that. The second type are more demanding and desires to learn more.

This introductory article with linked materials is to meet the need of the both types of readers. For the former type, I will suggest exactly the pages of the provided materials to read, while the latter one is encouraged to study based on the materials until his curiosity is fulfilled.

I have to apologize in advance for two things: a beautiful subject poorly introduced here and my time too limited to perfect everything presented.

1 Rock-paper-scissors

"Freedom is the freedom to say that two plus two makes four. If that is granted, all else follows."

— Nineteen Eighty-Four, George Orwell

Let us start with an example. Liangzu and Zuliang are playing the game Rock-paper-scissors, and Liangzu wants to always win. How should he cheat?

We need to understand the game first. The way for Laingzu to win is quite simple.

  • If Zuliang forms with his hand the shape rock, then Liangzu should give a paper.
  • If Zuliang forms with his hand the shape paper, then Liangzu should give a scissors.
  • If Zuliang forms with his hand the shape scissors, then Liangzu should give a rock.

This winning rule may be stated in a more concise way. 

if Zuliang forms a rock:
    Liangzu gives a paper
if Zuliang forms a paper:
    Liangzu gives a scissors
if Zuliang forms a scissors:
    Liangzu gives a rock

If then letting rock and paper and scissors form a set, say \(\{rock, paper, scissors \}\), then we know from the course that this set is isomorphic to \(\{1,2,3\}\). Simplifying what we have gives 

if Zuliang gives 1:
    Liangzu gives 2
if Zuliang gives 2:
    Liangzu gives 3
if Zuliang gives 3:
    Liangzu gives 1

If anybody thinks that the names in the above blok are too long to be handsome, I will shoot him. But he is correct. Let us denote by \(n\) and \(m\) the choices of Zuliang and Liangzu respectively. The lovely and warm block above becomes cold, or if you want, Python code. 

def rock_paper_scissors(n):
  if n == 1:
    m = 2
  if n == 2:
    m = 3
  if n == 3:
    m = 1
  return m

A few more things to explain. We start with Lines \(2\) - \(7\). Here n==1 reads "\(n\) equals to \(1\)" and similarly for n==2 and n==3, while by m=2 we mean that "let \(m\) have the value of \(2\)" or that "assign the value of \(2\) to \(m\)". In the first line, we defined a function named rock_paper_scissors. It takes an input \(n\), and asks whether \(n\) equals to \(1\), if yes then let \(m=2\). It continues to ask whether \(n\) equals to \(2\), if yes let \(m=3\). And so on. Finally in the line \(8\), this function returns the value of \(m\). In mathematical terms, rock_paper_scissors is merely an injective function from \(\{1,2,3\}\) to \(\{1,2,3\}\). Alternatively we may call this function \(f\) and we have \(f(1)=2,f(2)=3,f(3)=1\). Hence, what we do here is precisely transforming a mathematical function into a piece of Python function. And the Python function rock_paper_scissors that we wrote has another name, algorithm, an algorithm that helps Liangzu cheat, an algorithm in \(8\) lines of code. More seriously but still informally, algorithms take some input data, here \(n\), performs a finite sequence of commands, here Lines \(2\) - \(7\), and produces some output, \(m\), and at last terminates. By commands, we mean that the instructions which can be implemented in some programming languages such as Python, not the one that takes the stars to a girl, or to several girls.

To summarize this example, we have done the following things.

  1. We have a problem to solve, here helping Liangzu cheat.
  2. We know exactly how to solve this problem, here the way to win the game, and we can describe it in natural language.
  3. We realize that this corresponds to a computational procedure, leading us to implementing a Python function rock_paper_scissors.

1.1 Discussions

More generally, to know how to transform so as to solve our computational problem, we need to answer the following questions.

  1. What problem do we want to solve? this is relatively easy to answer. I want to sort a list of numbers. I want to perform a matrix-vector multiplication.
  2. What is the algorithm that solves our problem? In most cases there are existing algorithms for our tasks, and we have to study them.
  3. What does the computer provide to us for implementing our algorithm? Everybody knows that two plus two makes four. Now, I am giving you two numbers, \(a\) and \(b\), how could one use the computer to add \(a\) and \(b\)? This requires to know how to talk to the computer, even before which it requires to know its language. Here we will use Python as the language to communicate. And the below is the Python code to add two numbers. Note that Python have provided us the symbol \(+\) for addition. Then, the code add(2,2) will give \(4\), and please guess what is the returned result of add(414351,51454395). We also see that Python provides us a way to define functions such as rock_paper_scissors and add, and the way to use them, such as rock_paper_scissors(1) and add(2,2). It indeed offers more as we will see.
def add(a,b):
  c = a+b
  return c

c = add(2,2)
print(c)
d = add(414351,51454395)
print(d)

Since we have defined the function add, we may immediately use (run) it. The code c = add(2,2) reads: add two numbers, \(a=2\) and \(b=2\), and assign the result to \(c\), or store the result in \(c\). If you run this code segment in Python, you will be able to see that the value of \(c\), which is \(4\), is printed on the screen since we wrote print(c). And similar meaning for the last two lines. The next block presents the printed result. 

: 4
: 51868746

Go back to those three questions that we discussed above. Any programming activity involves answering them, implicitly or explicitly. This article is concerned with but the third one, talking to the computer (via Python). To be a better talker it further requires to change one's mindset into what we call computational thinking. Here is a comparison between mathematical and computational thinking.

  • In mathematics, one cares about the existence and uniqueness of certain objects, as well as the property of it.
  • If giving some problem to a computer scientist, his naturally asks how to do computation towards its solution.
  • They are not entirely isolated. It often happens that some constructive proof establishing the existence of some object corresponds to a computational procedure. QR decomposition is such an example.

The best way to cultivate computational thinking is via programming practice, for which we will arrange exercises whenever possible. The idea is that you do not need to know too much Python to solve problems. On the contrary, very few of it is quite enough. (Pioneers of computer science used to write programs for the machines which do not even exist).

2 Python basics

We assume that the reader has finished reading the materials recommended in the previous section, and has installed Python on the machine. The reader should also know what is Python interpreter (roughly) and how to run Python code in it.

The well-known slogan in Ministry of Truth (Newspeak: Minitrue) is the following. 

「War Is Peace; Freedom Is Slavery; Ignorance Is Strength.」

「戰爭即和平；自由即奴役；無知即力量。」

That is, Minitrue has defined the word War as Peace, and Freedom as Slavery, and Ignorance as Strength. That is, Ministrue assigns new semantics to War and Freedom and Slavery.

This section is concerned with some basic syntax and semantics about Python, that is, the language that Python (Minitrue) admits and its meaning. After learning them we are able to write a variety of Python programs for different purposes.

2.1 Python as a calculator

Python provides several symbols, + and - and * and /, for calculation. Those symbols behave as expected. 

>>> 2+2
4
>>> 1/3
0.3333333333333333
>>> 4-100
-96
>>> 8*9
72

When we write \(2+2\) we the Python interpreter computes the result of the \(2+2\), and returns the value \(4\).

Reading Materials: Section 2.2 of GCM/3rd.

2.2 Variables of numeric types

To achieve more than simple calculation, we need more devices. The first concept towards sophisticated programs is Variables. Variables allow us to store the intermediate computational results which can be used in what to come. 

>>> v = 3*3
>>> v+100
109
>>> v/4
2.25
>>> v-1
8
>>> b = 20
>>> v+b
29
>>> v-b
-11
>>> v = 4
>>> v
4
>>> v = v+2
>>> v
6

Based on what has been told so far, we know what the above fragment means. Here are more details. We defined a variable with the name v, and assign the value of \(3\times 3\) to it. Then we make some calculations and the results are as expected. Along the way we also define another variable with the name b and the value \(20\). After some computation we assign a new value, here \(4\), to v, after which, of course, the variable v has a new value and any computation about v is associated to its new value. Then we write v=v+2. This line of code involves two steps. First, compute the value of the expression v+2, which is \(v+2=4+2=6\). Then assign the value of \(6\) to the variable v. As a result, v as a new value after this sentence.

One important rule about wielding variables is that, they can only be used when they have been defined. Example: 

>>> c
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'c' is not defined

The last three lines are the error message given by Python. Looking at the last line where we can understand, Python complains that the name c is not defined.

Finally, it is important to have a good name for variables so that those who read the code immediately understand the role the variable plays in the program. We will illustrate this point when we are ready to compose larger programs. At current point it suffices to know that ordinary names will work as expected. Example: 

>>> War = 3
>>> Slavery = 4.99
>>> Ignorane = 88.8
>>> War
3
>>> Slavery
4.99
>>> Ignorane
88.8

2.3 Three constructs: sequential, conditional, iterative

So far in this section we have written some Python code only in one line. And from Section 6.1.2 of PP/3rd we know that there are three constructs that compose the program. Here we will learn how to express those constructs in Python. These three constructs will be the main tools that assist us to think computationally.

Sequential
We have met them before. Suppose we have defined two variables a0=0 and a1=1. We want to compute the fifth number a4 in the Fibonacci sequence, starting with a0. The code is as follows:
a0 = 0
a1 = 1
a2 = a1 + a0
a3 = a2 + a1
a4 = a3 + a2
print(a4)

One verifies that the above code does exactly what we want. Run this code here, and we will see that Python prints the value of a4, which is \(3\). The above code block gets excuted line by line, in agreement with our intuition. We note that each line is a statement. At a higher level the sequential construct looks like: 

<statement>
<statement>
<statement>
<statement>
<statement>
<statement>

Each statement corresponds to some computational (sub)-task. To be more precise, we write 

<statements>

That is, we use <statements> to represent a sequence of <statement>. In the words of PP/3rd, the computer is to carry out the first subtask completely, then go on and carry out the second subtask completely — never going back to the first subtask after starting the second subtask.

Conditional
We have met them before. Recall in the rock_paper_scissors function that we have used the conditional construct to deal with different shapes that Zuliang will form. Conditional construct is a way to deal with (binary) uncertainty. The baby may cry or not, it may rain or not, we may go back to school or not. When facing two choices we need to act correspondingly for each one. The conditional construct in Python looks like this:
if a == 1:
  b = 1
  c = 2
else:
  b = 0
  c = 3

Assuming a is some variable already defined somewhere, the above code justifies itself well. If the value of a is \(1\), then let b=1. Otherwise let b=0. And similarly for c. Special attention must be paid to the colons at the ends of lines 1 and 3. They are just more than the syntax of the if-else statment, which Python can recognize. Any (slight) violation to the syntax can not get excuted by Python. More generally, the syntax for if-else statement of Python looks like: 

if <b>:
  <statements>
else:
  <statements>

if <b>:
  <statements>

Here <b> can be thought of as a value which is either True or False (<b> for "binary"). For example, if we have written a=1, the statement a==1 will be evaluated by Python as True, and the value of the statment a==2 is False, since obviously \(1\neq 2\).

The second remark about the if-else construct is that, sometimes the else branch may be omitted (recall rock_paper_scissors).

Finally, in Python, and in almost all programming languages, the if-else construct is a statement. As a result, the above block is recursively defined, since the if-else statement that it can also arise in <statments> of the above block. More concretly, the following is a valid Python program, and is able to deal with more complicated tasks. 

if a == 1:
  if b == 3:
    c = 2
  else:
    if g = 203:
      c = 100
  d = 5
  e = 100
else:

Describe in your language what the above code is doing.

Remark. Do not worry about whether the assignment statements appearing in the above make sense, you will eventually replace, for example, c=2 by your own statements which do your job. Just remember our slogan: "Freedom is the freedom to say that two plus two makes four. If that is granted, all else follows". The if-else construct gives you some freedom, and everything else will follow. Here is an example. You want to write a Python program which replies the emails automatically. But your replies to your girl friend and father should be different. This is the code: 

if email_sender == "Wife":
  reply("Honey Honey!")
if email_sender == "Father":
  reply("Roger that!")

At some point in your career you will learn in Python how to define the variable email_sender based on the given information (e.g., you can know the email address which sends you the email), and reply.

Iterative
We have not met them in Python. But we have met them already in life. Liangzu is a robot worker in the virtual factory moving bricks from here to there. He has \(10000\) bricks to move. There are \(10000\) steps to move it. First, move the \(0\) -th brick. Second, move the \(1\) -st brick. Proceeding, and finally, move the \(9999\) -th brick. Correspondingly the code may be
move(brick[0])
move(brick[1])
...
move(brick[9999])

Would everyone like to write \(10000\) lines of code to simply move the bricks. We desire a simplification. Let \(I\) be the list of integer indices ranging from \(0\) to \(9999\), that is, \(I=[0,1,\dots, 9999]\). 

for each i in I
  move(brick[i])

Reading the above two blocks, they feel equivalent. Clearly, the latter is much simpler; to iteratively compute a similar thing we do not need to repeatedly write the code. That's the idea of the iterative construct. There is only one mere difference between above block and the for-loop statement in Python. 

for <variable> in <list>:
  <statements>
