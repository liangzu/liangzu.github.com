Introduction

C++ hasn't found its place in web applications, partly because there are no built-in string-operation libraries. You say what? std::string? Oh my god! std::string is something hard to use and sometimes leads a subtly buggy program.

This article introduces some conventional usages of std::string and tells people that don't be stupid with std::string.

Tips

we always want to do something interesting with strings. e.g.:

replace substr A with substr B in string X.

how we do this in some other programming languages? X.Replace(A, B);, right? that's all we need to code. but things are different in c++. Let's check the API of string, so many member functions but none of them are what I want! 

string& replace (size_t pos,        size_t len,        const string& str);
string& replace (const_iterator i1, const_iterator i2, const string& str);
string& replace (size_t pos,        size_t len,        const string& str,
                 size_t subpos, size_t sublen = npos);
string& replace (size_t pos,        size_t len,        const char* s);
string& replace (const_iterator i1, const_iterator i2, const char* s);
string& replace (size_t pos,        size_t len,        const char* s, size_t n);
string& replace (const_iterator i1, const_iterator i2, const char* s, size_t n);
string& replace (size_t pos,        size_t len,        size_t n, char c);
string& replace (const_iterator i1, const_iterator i2, size_t n, char c);
template <class InputIterator>
string& replace (const_iterator i1, const_iterator i2,
                   InputIterator first, InputIterator last);
string& replace (const_iterator i1, const_iterator i2, initializer_list<char> il);

And then, we try to find our own solution. the pseudocode may simply looks like: 

len = A.size()
pos = X.find(A)
X.replace(pos, len, B);

OK! the size of A is something A itself already knew. our problem becomes:

How to find A in X.

There seems a built-in find function in std::string, let's have a look! 

size_t find (const string& str, size_t pos = 0) const noexcept;
size_t find (const char* s, size_t pos = 0) const;
size_t find (const char* s, size_t pos, size_type n) const;
size_t find (char c, size_t pos = 0) const noexcept;

luckily, there are something what we need: the first one. Now we can develop our own replace function: 

void replace_first(string X, string A, string B)
{
  X.replace(X.find(A), A.size(), B);
}

looks verbose, right? That's the way c++ looks like. Its aim is not providing high-level operation and let the programmer code just like a API caller. Nevertheless, C++ provides you the room to develop your own abstraction with a little bit effort.

We can also insert substr A before/behind substr B with the combination of built-in string::insert and string::find.

Maybe you've found also there aren't join, spite operations in std::string, you can implement you own one by using templates.

Traps

Sometimes we may want to build our own types, with std::string serving as a member variable: 

struct MyType
{
    std::string str;
};

Of course constructors are needed, how do we implement our constructors? there is an intuitive way: 

struct MyType
{
    std::string str;

    MyType(const char* s)
        :
        str{s}
    {}
};

This program is plausible, or seriously, it's wrong!!! what happens if we construct MyType with a stupid argument nullptr? 

void PossibleUsage()
{
    auto t = MyType(nullptr);
}

The compiler won't catch this situation, because const char* as an argument is legal for std::string and the type of nullptr is a pointer(maybe void*?). This program, however, will crash in runtime, with an exception ignored: 

terminate called after throwing an instance of 'std::logic_error'
  what():  basic_string::_M_construct null not valid

std::string drops the responsibility of checking the illegal argument to the user. I don't know if it's a good design or not. But we should always check the pointers 

void PossibleUsage()
{
    const char* s = DoSomething();

    if (s != nullptr) {
        auto t = MyType(s);
    }
}

And, sometime we can also write constructor like this: 

struct MyType
{
    std::string str;

    MyType(const char* s)
    {
        if (s == nullptr) {
          return; 
        }
        else {
            str = s;
        }
    }
};

Conclusion

  • Be smart with =std::string.
  • Be aware of pointers: checking before using.
  • Understand std::string and its pal: (const) char*.
