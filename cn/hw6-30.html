简介

本次作业是对给定的语法做类型检查，我的基本思路是:

  • 用Flex, Bison以及自己的数据结构构建语法树.
  • 收集变量声明(VarDecl)信息，构建类型环境(TypeEnv)的符号表.
  • 自底向上、后序遍历语法树，进行类型检查.

下面也将分这三个部分进行介绍。

%token ID INT BOOL ARRAY OF IF THEN BOOL_CONST INT_CONST ASSIGN OR

%type <TokenType> ID INT BOOL ARRAY OF IF THEN BOOL_CONST INT_CONST ASSIGN OR

%type <ProgramPtr> program

%type <VarDeclPtr>     var_decl
%type <VarDeclList>    var_decls

%type <TypeExprPtr>    type_exp

%type <StatementPtr>   stmt
%type <StatementList>   stmts

%type <ExprPtr>        exp


%left OR
%left PLUS



%%


program:
  var_decls ';' stmts
    { root = new Program($1, $3); }

var_decls:
  var_decls ';' var_decl
    { $1->push_back($3); $$ = $1; }
  | var_decl
    { $$ = new List<VarDecl>(); $$->push_back($1); }

var_decl:
  ID ':' type_exp
    { $$ = new VarDecl($1, $3); }

type_exp:
  INT
    { $$ = new IntType(); }
  | BOOL
      { $$ = new BoolType(); }
  | ARRAY '[' INT ']' OF type_exp
      { $$ = new ArrayType($3, $6); }


stmts:
  stmts ';' stmt
    { $1->push_back($3); $$ = $1; }
  | stmt
      { $$ = new List<Statement>(); $$->push_back($1); }
stmt:
  IF exp THEN stmt
    { $$ = new If($2, $4); }
  | ID ASSIGN exp
    { $$ = new Assign($1, $3); }

exp:
  exp '+' exp
    { $$ = new Plus($1, $3); }
  | exp OR exp
    { $$ = new Or($1, $3); }
  | ID '[' exp ']'
    { $$ = new ArrayExpr($1, $3); }
  | INT_CONST
    { $$ = new IntConst($1); }
  | BOOL_CONST
    { $$ = new BoolConst($1); }
  | ID
    { $$ = new Identifier($1); }

构建类型环境

由于该文法没有类似于嵌套作用域的变量声明，所以我们只需要建立一个全局的类型环境。稍微具体一点就是：

对于每一个变量声明结点，将该结点的变量名作为 key,类型作为 val,插入符号表。

构建类型环境的代码如下: (in tree.h) 

TypeEnv BuildTypeEnv()
{
    TypeEnv typeEnv;   // 类型环境

    for (auto varDecl: *varList)
    {
        auto id = varDecl->id;                     // 获取变量名称
        auto typeName = varDecl->type->TypeName(); // 获取类型名称

        if (typeEnv.find(id) == typeEnv.end())
        {
            typeEnv[id] = typeName;             //如果不存在该变量， 则插入。
        }
        else
        {
            printf("repeatly defined variables!!\n");  // 如果已存在，则说明重复定义。
        }
    }
    return typeEnv;
}

类型检查

我的做法是使用每个类里面都存在的成员函数: virtual string TypeChecker(TypeEnv tv), 该函数接受当前类型环境作为参数，对它所在 的类(class)进行类型检查。返回值是这个类的类型属性（例如，表示integer相加的 Plus表达式 类应该返回一个整数类型)。

bool, int常量

由于它们是‘独立’的类型(bool 和 int)，所以直接返回它们的类型属性就可以了。 

struct IntConst: Expr
{
    int val;

    virtual string TypeChecker(TypeEnv tv)
    {
        return "int";
    }
};

这是整数常量(IntConst)的代码，布尔常量(BoolConst)也是类似的。

identifier

identifier的类型信息可以通过之前建立的类型环境(TypeEnv)查得,如果不存在这个identifier则会报错。 

virtual string TypeChecker(TypeEnv tv)
{
    if (tv.find(id) == tv.end())
    {
        printf("undeclared variable reference in Assignment !!\n");
        return "nil";
    }
    return tv[id];
}

加法表达式和Or表达式

先对它们的子表达式进行类型检查，再检查它们自己。 

struct Plus: Expr
{
    Ptr<Expr> left;          // '+'号左边
    Ptr<Expr> right;         // ‘+’号右边

    virtual string TypeChecker(TypeEnv tv)
    {
                                                   // 后序遍历
        auto leftType = left->TypeChecker(tv);
        auto rightType = right->TypeChecker(tv);
        if (leftType == "int" && rightType == "int")
        {
            return "int";             
        }
        else
        {
            printf("different type at '+' expression");
            return "nil";                           //报错
        }
    }
};

Or表达式也是类似的。

数组

数组引用包括数组名(identifier)和下标，由于文法没有说num是什么，我直接认为它是整数了。 数组名对应的数组类型已经保存在了类型环境中。 

struct ArrayExpr: Expr
{
    string arrayName;
    Ptr<Expr> index;

    virtual string TypeChecker(TypeEnv tv)
    {
        auto indexType = index->TypeChecker(tv);

        if (tv.find(arrayName) == tv.end())
        {
            printf("undefined identifier in array reference!!\n");
        }
        else if (indexType != "int")
        {
            printf("array subscript must be a integer!!\n");
            return "nil";
        }
        else return tv[arrayName];
    }
};

赋值状态

这里需要检查的是等号两端类型一致以及左边的identifier在类型环境里有定义。 

struct Assign: Statement
{
    string id;
    Ptr<Expr> expr;

    virtual string TypeChecker(TypeEnv tv)
    {
        auto exprType = expr->TypeChecker(tv);
        if (tv.find(id) == tv.end())
        {
            printf("undeclared variable reference in Assignment !!\n");
        }
        if (exprType != tv[id])
        {
            printf("types are not compatible in Assignment");
        }
        return "nil";
    }
};

If状态

这里需要检查的是条件判断语句是 bool ,以及它的状态语句(stmt)。 

struct If: Statement
{
    Ptr<Expr> cond;
    Ptr<Statement> stmt;

    virtual string TypeChecker(TypeEnv tv)
    {
        auto condType = cond->TypeChecker(tv);
        auto stmtType = stmt->TypeChecker(tv);

        if (condType != "bool")
        {
            printf("non-boolean expression in if condition");
        }
        return "nil";
    }
};

由于文法没指定状态语句应该返回什么类型，所以我在这里随便搞了一下。

Program

由于program由变量声明list和状态list组成，前者用于构建类型环境，我们只要对所有的stmt list进行类型检查就可以了。 

virtual string TypeChecker(TypeEnv tv)
{
    if (stmtList == nullptr)
        return "nothing to check";

    for (auto stmt: *stmtList)
    {
        stmt->TypeChecker(tv);
    }
}
